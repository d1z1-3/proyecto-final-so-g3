Desarrollo de los Componentes Técnicos

Componente 1: Virtualización con Linux

Despliegue de dos entornos virtuales independientes utilizando la distribución asignada (Fedora 41).

  • VM 1: Fedora 41 (Entorno Gráfico de Escritorio).
  • VM 2: Fedora 41 Server (Entorno de Consola Pura).
  • Particionamiento: Configuración manual del disco con las áreas requeridas: /, swap y /home.
  • Red: Enlace inter-VM mediante adaptador de red interna o puente, habilitando el acceso remoto seguro vía SSH de la VM gráfica a la de consola.

Componente 2: Contenedores Docker

Aislamiento de servicios locales mediante la instalación y aprovisionamiento de Docker Engine en la máquina virtual gráfica.

  • Frontend: Servidor web Nginx que aloja la estructura estática de este sitio web de documentación.
  • Backend: Servidor básico HTTP en Python configurado y expuesto en el puerto 5000.
  • Orquestación: Automatización del despliegue unificado y gestión de volúmenes persistentes con Docker Compose utilizando como imagen base fedora:41.

Componente 3: Orquestación con Kubernetes

Configuración del cluster local de un solo nodo mediante Minikube en el S.O. gráfico de Fedora para la gestión automatizada de las cargas de trabajo.

  • Manifiestos: Definición lógica de la arquitectura en archivos YAML (Deployment y Service tipo NodePort).
  • Réplicas: Despliegue inicial de la imagen oficial de Nginx parametrizada con 2 réplicas y escalado manual verificado a 3 réplicas operativas.

Evidencias de la Entrega 2 (Capturas):

Verificación del estado de los Pods y el Servicio expuesto:

kubectl get pods
kubectl get svc

Escalado de la infraestructura a 3 réplicas activas:

kubectl scale deployment/nombre-deployment --replicas=3

Conclusiones del Proyecto

  • El particionamiento manual asegura un aislamiento lógico del almacenamiento, impidiendo que desbordamientos en directorios de usuarios (/home) comprometan la estabilidad del sistema raíz (/) en Fedora 41.
  • Docker Engine permite la ejecución eficiente de microservicios sin la penalización de rendimiento de la virtualización completa, garantizando entornos idénticos de desarrollo tanto para el servidor Python como para Nginx.
  • Kubernetes (a través de Minikube) automatiza la tolerancia a fallos de la capa web mediante políticas de replicación, facilitando pautas de escalado horizontal rápido ante variaciones de carga de trabajo.
