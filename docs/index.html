Desarrollo de la Infraestructura

Componente 1: Virtualización con Linux

Despliegue de dos entornos virtuales independientes utilizando la distribución asignada (Fedora 44).

  • VM 1: Fedora 44 (Entorno Gráfico de Escritorio).
  • VM 2: Fedora 44 Server (Entorno de Consola Pura).
  • Particionamiento: Configuración manual del disco con las áreas requeridas: /, swap y /home.
  • Red: Enlace inter-VM mediante adaptador de red interna, habilitando el acceso remoto seguro vía SSH de la VM gráfica a la de consola.

Evidencias de Implementación:

Particionamiento manual y asignación de discos:

Prueba de conectividad inter-VM y acceso SSH funcional:

[dajo12@fedora ~]$ ssh usuario_servidor@ip_maquina_consola

Componente 2: Contenedores Docker

Aislamiento de servicios locales mediante la instalación y aprovisionamiento de Docker Engine nativo en la máquina virtual gráfica.

  • Frontend: Servidor web Nginx que aloja la estructura estática del sitio.
  • Backend: Servidor básico HTTP en Python configurado y expuesto en el puerto 5000.
  • Orquestación: Automatización del despliegue unificado y gestión de volúmenes persistentes con Docker Compose utilizando como imagen base fedora:44.

Evidencias de Implementación:

Despliegue exitoso de contenedores mediante Compose:

[dajo12@fedora ~]$ docker compose up -d

Validación de los servicios en el navegador web:

Componente 3: Orquestación con Kubernetes

Configuración del cluster local de un nodo mediante Minikube para la gestión automatizada de alta disponibilidad.

  • Manifiestos: Definición lógica de la arquitectura en archivos YAML (Deployment y Service tipo NodePort).
  • Réplicas: Despliegue inicial de la imagen oficial de Nginx parametrizada con 2 réplicas y escalado manual verificado a 3 réplicas operativas.

Evidencias de Implementación:

Verificación del estado de los Pods y el Servicio expuesto:

[dajo12@fedora ~]$ kubectl get pods
[dajo12@fedora ~]$ kubectl get svc

Escalado de la infraestructura a 3 réplicas activas:

[dajo12@fedora ~]$ kubectl scale deployment/nginx-deployment --replicas=3

Conclusiones del Proyecto

  • El particionamiento manual asegura un aislamiento lógico del almacenamiento, impidiendo que desbordamientos en directorios de usuarios (/home) comprometan la estabilidad del sistema raíz (/) en Fedora 44.
  • Docker Engine permite la ejecución eficiente de microservicios sin la penalización de rendimiento de la virtualización completa, garantizando entornos idénticos de desarrollo tanto para el servidor Python como para Nginx.
  • Kubernetes (a través de Minikube) automatiza la tolerancia a fallos de la capa web mediante políticas de replicación, facilitando pautas de escalado horizontal rápido ante variaciones de carga de trabajo.
